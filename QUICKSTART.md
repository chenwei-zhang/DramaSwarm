## 一、DramaSwarm 是什么？

beta_e

DramaSwarm 是一个**娱乐圈危机公关仿真器**。它包含三套仿真引擎：
beta_e

1. **知识图谱引擎** — 存储明星关系、八卦事件，用 D3.js 可力导向图交互展示
beta_e
2. **危机仿真引擎** — 模拟公关危机从爆发到平息的全过程，用户可干预看what-if" 推演
beta_e
3. **综艺修罗场引擎** — 模拟综艺节目中的群体互动

Beta_e

**核心理思想**：每个明星是一个 Agent，根据收集到的信息定制化其行为模式，然后明星之间交互，用户可以干预观察不同策略的效果。

beta_e

**不需要 LLM API Key**： 所有功能基于纯规则引擎，启动即装 `pip install -r requirements.txt` 即可运行。

**需要 LLM 时**： 配置 `.env` 文件中的 API Key，Agent 会生成更自然的对话，但不影响核心功能。

beta_e

**知识图谱数据来源**：`celebrity_scraper/` 目录下的 JSON 文件（已内置 10 位明星数据）

也可运行爬虫添加更多。

 Currently包含 10 位明星：杨幂、 肖战、 李小璐、 贾乃亮、 PG One、 唐嫣、 襄阳、 赵丽颖

## 四、危机模拟 Tab
beta_e

### 4.1 核心概念
beta_e

模拟一个娱乐圈危机事件从爆发到平息的全过程。你扮演"上帝视角"设定干预条件，观察不同策略的效果。

**每个明星是一个 Agent**，拥有：
- **10 维人格画像**： 从知识图谱数据推导（职业/粉丝/作品/争议历史等）
- **口碑分**： 0-100，越高越好，但期间品牌价值、热度、监管等级随之变化
- **关系影响**: Agent 之间根据关系类型（配偶/对手/搭档）互相影响决策
- **观众反应**: 30 个观众 Agent 生成评论，影响明星策略
- **干预系统**: 在仿真前或中添加干预，自动触发

- **结果分析**: 对比仿真结果与历史基线

### 4.2 危机生命周期
β_e

危机从爆发到平息经历 6 个阶段，每个阶段的持续时间和特点不同，Agent 的决策策略不同。

| 鶰 天数 | 阶段 | 持续天数 | 特点 |
|------|------|------|
| Day 1 | 爆发期 BREAKOUT | 事件刚曝光，舆论一片哗然。**最佳策略：发声明澄清** |
| Day 2-3 | 发酵期 ESCALATION | 细节流出、谣言扩散。**沉默变得危险**， |
| Day 4-7 | 高峰期 PEAK | 全网讨论，最高关注度。**道歉/上节目最佳窗口** |
| Day 8-14 | 应对期 MITIGATION | PR 策略见效或各方表态。**公益/复出可行** |
| Day 15-21 | 收尾期 RESOLUTION | 降温。**最安全的公益/复出时机** |
| Day 22+ | 余波期 AFTERMATH | 长期影响。**复出试水** |

> **关键规律**: 高峰期（Day 4-7）是道歉效果最好（×1.2），余波期道歉几乎没意义（×0.3)。爆发期道歉反而显得心虚（×0.3)。

### 4.3 操作指南
β_e

#### Step 1: 选择场景并启动
β_e

**方式一** — 弯览史场景
: 左上角"场景选择"下拉，选中一个历史危机场景，点击 **"启动仿真"**。

**方式二** — 自由组局: 点击"自定义场景"按钮，自己选择明星创建场景。

#### Step 2: 推进仿真
beta_e

| 按钮 | 效果 |
|------|------|
| **下一步 (1天)** | 逐天推进，观察每天变化 |
| **运行全部** | 一次性跑完 30 天 |
| **查看结果** | 对比仿真与历史基线 |

#### Step 3: 观察面板
β_e

仿真过程中可以实时观察多个面板:

| 面板 | 内容 | 作用 |
|------|------|------|
| 人物状态 | 每人的口碑分（0-100）、品牌价值、监管等级 | 口碑分是公众好感度指标，直接影响商业价值。品牌价值低→品牌可能解约 |
| 口碑走势图 | Canvas 折线图，每人一条线 + 热度虚线 | 直观展示口碑变化趋势 |
| 事件时间线 | 每天的 Agent 动作、谣言、触发关系 | 了解每天发生了什么 |
| 观众反应 | 正面/负面/中性评论 | 观察公众情绪走向 |
| 热搜榜 | 排名 + 热度值 | 监控舆论焦点 |
| 媒体头条 | 各媒体的标题和立场 | 了解媒体报道角度 |
| 品牌状态 | 代言品牌的动作（继续/观望/暂停/解约） | 预测商业风险 |
| 结果对比 | 仿真 vs 历史基线 | 了解策略效果 |

#### Step 4: 查看结果
β_e

点击 **"查看结果"**，系统给出：
- **Verdict**（更好/更差/相似）: 仿真结果与历史真实结局的整体对比
- **指标对比**: 每人的最低/最终口碑、最高热度、谣言数等
- **PR 建议**: 基于规则的公关策略建议

## 五、干预操作
β_e

**核心功能**: 在仿真中添加"what-if"干预条件，观察不同策略的效果。

### 5.1 使用场景
beta_e

- 想测试"如果杨幂在第 3 天道歉会怎样？" → 添加一个"第 3 天让杨幂道歉"的干预
- 想测试"注入一个视频泄露会有什么影响？" → 添加一个"注入视频泄露"的干预
- 想测试"改变两人关系会怎样？" → 添加一个关系变更干预

### 5.2 操作步骤
β_e

1. **先启动仿真**（干预必须在仿真启动后添加）
2. 在干预面板选择触发类型：
   - **定时触发**: 在第 X 天触发（如"第 5 天"）
   - **相对天数**: 仿真开始后 N 天触发
   - **状态阈值**: 某指标达到阈值时触发
3. 选择动作类型：
   - **强制动作**: 让某人执行指定 PR 策略
   - **注入事件**: 6 种外部事件（媒体报道/视频泄露/对手声明/监管行动/品牌决策/自定义）
   - **关系变更**: 修改两人间的关系强度
4. 填写参数并点击 **"添加干预"**
5. 干预出现在"待触发"列表中
6. 推进仿真时自动触发

### 5.3 三种触发类型

| 类型 | 说明 | 使用时机 |
|------|------|---------|
| 定时触发 | 在指定天数精确触发 | "第 5 天让杨幂道歉" |
| 相对天数 | 仿真开始后 N 天触发 | "3 天后注入视频泄露" |
| 状态阈值 | 口碑/热度/品牌/监管达到阈值时触发 | "当杨幂口碑 < 30 时触发票白" |

> **状态阈值示例**: 选择指标=口碑、比较=<、阈值=30、人物=杨幂。 当杨幂的口碑跌破30 时自动触发。
此时可以选择"触发后执行什么动作（道歉/声明/公益等），从而实现自动响应式干预。

### 5.4 三种动作类型

| 类型 | 说明 | 效果 |
|------|------|------|
| 强制动作 | 指定某人执行某个 PR 策略 | 直接改变该人的行为，跳过 Agent 自动决策 |
| 注入事件 | 6 种外部事件 | 影响口碑、热度、品牌价值，产生配套热搜和媒体报道 |
| 关系变更 | 修改两人间的关系 | 加强/减弱/断裂/新建，影响后续交互 |

### 5.5 注入事件的 6 种类型

| 事件 | 口碑影响 | 热度影响 | 适用场景 |
|------|---------|---------|---------|
| 媒体报道 | -5 | +15 | 主流媒体发布深度报道 |
| 视频泄露 | -10 | +25 | 关键证据被曝光（最大冲击） |
| 对手声明 | -3 | +10 | 猜争对手发声 |
| 监管行动 | -8 | +20 | 政府部门介入 |
| 品牌决策 | -4 | +8 | 代言品牌做出决策 |
| 自定义 | -5 | +10 | 自定义事件 |

> **示例**: 添加"视频泄露"事件 → 选"注入事件" + 事件类型="视频泄露" + 事件描述="监控视频流出"。 会自动降低口碑 10 分、推高热度 25 分、并生成热搜和媒体报道。

### 5.6 关系变更的 4 种操作

| 操作 | 说明 | 效果 |
|------|------|------|
| 加强 | 关系强度 +0.2 | 双方合作更紧密 |
| 减弱 | 关系强度 -0.2 | 互信下降 |
| 断裂 | 关系强度 → 0.05 | 关系几乎断裂 |
| 新建 | 创建新关系 | 建立新的连接 |

### 5.7 10 种 PR 策略详解

β_e

| 猛略 | 说明 | 最佳时机 | 效果倍率 |
|------|------|---------|---------|
| 沉默不回应 | 不做任何回应 | 热度低、口碑高时 | 热度每天 +5、谣言概率 ×1.3 |
| 公开道歉 | 承认错误请求原谅 | 高峰期 | 效果 ×1.2（爆发期 ×0.3，太早道歉反显心虚） |
| 发声明 | 官方澄清 | 发酵期 | 效果 ×1.0 |
| 上节目回应 | 正面回应争议 | 应对期 | 效果 ×1.1 |
| 起诉造谣者 | 法律维权 | 应对期之后 | 效果 ×1.0 |
| 卖惨博同情 | 展示脆弱面 | 高峰期 | 效果 ×1.0 |
| 反击否认 | 强力否认指控 | 高风险高回报 | 效果因口碑而异 |
| 暂避风头 | 集退避开公众 | 任何时候 | 热度 +3、谣言概率 ×1.5 |
| 公益洗白 | 参与公益活动 | 收尾/余波期 | 效果 ×1.3（最佳公益时机） |
| 复出试水 | 发布新作品试水 | 余波期 | 效果 ×1.3（最佳复出时机） |

> **重要**: 篝略阶段修正器是关键。 高峰期道歉效果 ×1.2，但爆发期只有 ×0.3。 余波期公益和复出效果 ×1.3。 选对时机比选对策略更重要。

>

### 5.8 操作示例

**场景**: 李小璐 PG One 夜宿门事件
**目标**: 测试"如果让李小璐在第 5 天道歉"会怎样"
**步骤**:
1. 启动仿真
2. 干预面板选: 定时触发 → 第 5 天
3. 动作类型: 强制动作 → 人物选李小璐 → 策略选公开道歉
4. 点击"添加干预" → 列表显示 "⏳ 第5天 | 李小璐→道歉"
5. 逐天推进仿真到第 5 天 → 干预自动触发 → 李小璐执行道歉
6. 观察口碑走势图的变化

## 七、A/B 对照实验
β_e

**核心价值**: 同一危机场景、不同干预策略的效果对比，找出最优方案。

### 7.1 使用场景
β_e

- "道歉和公益哪个更有效?" → 对比"道歉组"和"公益组"
- "沉默 vs 反击哪个风险更大?" → 对比"沉默组"和"反击组"
- "早期干预 vs 晚期干预?" → 对比"第 3 天干预组"和"第 10 天干预组

### 7.2 操作步骤

β_e

1. 危机 Tab 底部点击 **"显示/隐藏"** 展开实验面板
2. 从下拉菜单选择一个危机场景
3. 添加实验组:
   - 默认有"对照组"（不添加干预）
   - 点击 "+" 添加更多组（如"道歉组"、"公益组"）
   - 至少需要 2 组
4. 点击 **"创建实验"**
5. 点击 **"运行"**（依次执行每组仿真）
6. 查看对比结果

### 7.3 对比结果说明
β_e

| 内容 | 说明 |
|------|------|
| 柱状图 | 各组平均口碑对比，一目了然 |
| 对比表 | 组名 / 平均口碑 / 最低口碑 / 热度 / 监管等级 |
| 口碑差值 | 各实验组相对于对照组的每人口碑变化（绿色=改善，红色=恶化） |
| 最佳策略 | 系统推荐最佳组 |

> **示例**: 选"夜宿门"场景，创建 3 组：
- 对照组（无干预）→ 平均口碑 35
- 道歉组（第 3 天道歉）→ 平均口碑 52（+17）
- 公益组（第 10 天做公益）→ 平均口碑 48（+13）
→ 结论: 道歉策略最优，系统推荐"道歉组"

## 九、命令行 Demo
β_e

不用启动 Web 服务，直接命令行体验核心功能。

### 9.1 危机仿真（纯规则，无需 API Key）
β_e

```bash
python -m demos.crisis_simulation
```

无需 API Key，无需联网。

### 9.2 综艺修罗场模拟（纯规则）

## 十、常见问题

β_e

**Q: 需要 API Key 吗？**
不需要。核心功能（危机仿真、自由互动、A/B 实验）全部基于规则引擎。 配置 LLM API Key 后 Agent 决策更自然，但不影响核心功能。
只有综艺 LLM 模式需要 API Key。

**Q: 为什么图谱只有 10 个明星？**
数据来自 `celebrity_scraper/data/` 目录下的 JSON 文件（10 个）可以运行爬虫添加更多：
```bash
python test_scraper.py real 肖战
python test_scraper.py real 杨幂
```

**Q: 自由互动和危机模式有什么区别？**
危机模式有 6 阶段生命周期，爆发→发酵→高峰→应对→收尾→余波），Agent 选择 10 种 PR 策略。自由互动没有阶段，Agent 选择 11 种日常动作（发言/支持/批评/合作等).自由模式更像沙盒，危机模式更像推演。

 crisis 有明确终点。

**Q: A/B 实验至少需要几组？**
至少 2 组（对照组 + 至少 1 个实验组）。建议 3-4 组。组数越多对比越有意义。

**Q: 干预什么时候触发？**
干预在仿真推进到对应天数时自动触发。 定时触发精确匹配天数； 相对天数从仿真开始算起； 绑状态阈值当指标达到阈值时触发。

**Q: 如何调试？**
```bash
# 查看完整仿真日志
python -m demos.crisis_simulation 2>&1 | tee sim.log

# 测试单个 API
curl -s http://localhost:8765/api/crisis/scenarios | python -m json.tool
```

**Q: 结果中的"更好"说明什么意思?**
仿真口碑普遍高于历史基线线，说明干预策略有效，实际值得借鉴。
 "更差"说明干预使情况恶化， 需要调整策略。 "相似"说明结果接近历史， 有参考价值但不突出。
