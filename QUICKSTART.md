# DramaSwarm 快速上手指南

> 从安装到完整操作的 Step-by-Step 指南

---

## 一、环境准备

### 1.1 安装依赖

```bash
cd /path/to/pr
pip install -r requirements.txt
```

### 1.2 配置 API Key（可选）

DramaSwarm 的核心功能（危机仿真、自由互动、A/B 实验）全部基于规则引擎，**不需要任何 API Key**。

如果想让 Agent 使用 LLM 生成更自然的对话，创建 `.env` 文件：

```bash
cp .env.example .env
# 编辑 .env，填入 GEMINI_API_KEY
```

### 1.3 验证安装

```bash
pytest tests/ -v    # 应显示 63 passed
```

---

## 二、启动服务

```bash
python run_viz.py
```

浏览器打开 **http://localhost:8765**

可选参数：
```bash
python run_viz.py --port 3000      # 自定义端口
python run_viz.py --reload          # 开发模式，代码改动自动重载
```

## 三、知识图谱 Tab

### 3.1 浏览关系网络

页面中央是一个 D3.js 力导向图，展示明星关系网络：
- **圆形节点** = 名人（蓝色）
- **菱形节点** = 八卦事件（橙色）
- **连线** = 关系（颜色区分类型：配偶/绯闻/搭档/对手/好友…）

操作：
| 操作 | 效果 |
|------|------|
| 鼠标拖拽节点 | 移动节点位置 |
| 滚轮 | 缩放视图 |
| 点击节点 | 右侧显示详情（关系列表、关联事件、Agent 上下文） |
| 点击空白处 | 取消选中，恢复全部显示 |

### 3.2 搜索名人

顶部搜索栏输入名字（如"杨幂"），回车即可定位并放大到该节点。

### 3.3 查找关系路径

顶部"路径"区域：选择起点和终点 → 点击"查找" → 图中高亮显示两人之间的完整关系链。

例如：`杨幂 → 赵丽颖` 可能显示：`杨幂 → 竞争对手 → 赵丽颖`

### 3.4 过滤视图

顶部"过滤"下拉可切换：
- **名人关系** — 只显示名人节点和关系边
- **全部节点** — 包含事件、新闻等所有节点

## 四、危机模拟 Tab

### 4.1 方式一：选择历史场景

这是最简单的上手方式：

**Step 1** — 选择场景

左上角"场景选择"下拉，系统从知识图谱中加载了 10 个历史危机场景：

| 场景 | 涉及人物 |
|------|---------|
| 李小璐PG One夜宿门事件 | 李小璐、贾乃亮、PG One |
| 杨幂刘恺威离婚 | 杨幂 |
| 赵丽颖冯绍峰离婚 | 赵丽颖 |
| 唐嫣凭《繁花》事业翻红 | 唐嫣 |
| ...更多 | ... |

**Step 2** — 启动仿真

点击 **"启动仿真"** 按钮。

**Step 3** — 推进仿真

- **"下一步 (1天)"** — 逐天推进，观察每天的变化
- **"运行全部"** — 一次性跑完 30 天

**Step 4** — 观察变化

仿真过程中可以实时观察：

| 面板 | 内容 |
|------|------|
| 人物状态 | 每人的口碑分、品牌价值、监管等级 |
| 口碑走势图 | Canvas 折线图，每人一条线 + 热度虚线 |
| 事件时间线 | 每天的 Agent 动作、谣言、触发关系 |
| 观众反应 | 正面/负面/中性评论 |
| 热搜榜 | 排名 + 热度值 |
| 媒体头条 | 各媒体标题 |
| 品牌状态 | 代言品牌的动作（继续/观望/暂停/解约） |

**Step 5** — 查看结果

点击 **"查看结果"**，系统对比仿真结果与历史真实结局，给出评价（更好/更差/相似）和 PR 建议。

### 4.2 方式二：自由组局

自己选择明星，创建自定义危机场景：

**Step 1** — 展开"自定义场景"

点击左侧面板的 **"自定义场景"** 按钮。

**Step 2** — 选择明星

从列表中勾选明星（最多 6 人），每人显示职业和粉丝数。

**Step 3** — 配置参数

- 场景标题（如"出轨风波"）
- 场景描述（可选）
- 八卦类型：出轨/丑闻/离婚/吸毒/偷税漏税/其他
- 严重度：0.1 ~ 1.0 滑块

**Step 4** — 发现关系

点击 **"发现关系"**，系统从知识图谱中查询选中明星之间的已有关系，如：

```
杨幂 — 竞争对手 — 赵丽颖
杨幂 — 好友 — 唐嫣
```

**Step 5** — 创建并启动

点击 **"创建并启动"**，系统自动创建场景、启动仿真。

## 五、干预操作

仿真运行中或启动前，都可以添加干预条件。

### 5.1 触发类型

| 类型 | 说明 | 示例 |
|------|------|------|
| 定时触发 | 在第 X 天触发 | "第 5 天让杨幂道歉" |
| 相对天数 | 仿真开始后 N 天触发 | "3天后注入视频泄露" |
| 状态阈值 | 某指标达到阈值时触发 | "当杨幂口碑 < 30 时触发票白" |

### 5.2 动作类型

| 类型 | 说明 |
|------|------|
| 强制动作 | 指定某人执行某个 PR 策略（10 种可选） |
| 注入事件 | 6 种外部事件：媒体报道/视频泄露/对手声明/监管行动/品牌决策/自定义 |
| 关系变更 | 加强/减弱/断裂/新建两人间的关系 |

### 5.3 操作步骤

1. 干预面板中选择触发类型
2. 选择动作类型（强制动作/注入事件/关系变更）
3. 填写具体参数（人物、动作、日期等）
4. 点击 **"添加干预"**
5. 干预条件出现在"待触发"列表中
6. 推进仿真时自动触发

### 5.4 10 种 PR 策略

| 策略 | 说明 | 最佳时机 |
|------|------|---------|
| 沉默不回应 | 不做任何回应 | 热度低、口碑高时 |
| 公开道歉 | 承认错误请求原谅 | 高峰期（效果 ×1.2） |
| 发声明 | 官方澄清 | 发酵期（效果 ×1.0） |
| 上节目回应 | 正面回应争议 | 应对期（效果 ×1.1） |
| 起诉造谣者 | 法律维权 | 应对期之后 |
| 卖惨博同情 | 展示脆弱面 | 高峰期 |
| 反击否认 | 强力否认指控 | 高风险高回报 |
| 暂避风头 | 隐退避开公众 | 任何时候 |
| 公益洗白 | 参与公益活动 | 收尾/余波期（效果 ×1.3） |
| 复出试水 | 发布新作品 | 余波期（效果 ×1.3） |

## 六、自由互动模式

除危机模式外，还可以让明星自由互动（11 种动作）：

| 动作 | 说明 |
|------|------|

## 七、A/B 对照实验

对比不同干预策略的效果：

**Step 1** — 展开实验面板

危机 Tab 底部点击 **"显示/隐藏"** 按钮。

**Step 2** — 选择场景

从下拉菜单选择一个危机场景。

**Step 3** — 添加实验组

- 默认有一行"组名"输入框
- 输入组名（如"对照组"）
- 点击 **"+"** 按钮添加更多组（如"道歉组"、"公益组"）
- 至少需要 2 组

**Step 4** — 创建并运行

- 点击 **"创建实验"**
- 点击 **"运行"**（依次执行每组仿真）

**Step 5** — 查看对比

运行完成后显示：
- 柱状图：各组平均口碑对比
- 对比表：组名 / 平均口碑 / 最低口碑 / 热度 / 监管等级
- 口碑差值：各实验组相对于对照组的每人口碑变化
- 最佳策略推荐

## 九、API 参考

所有端点均以 `http://localhost:8765/api/` 为前缀：

### 图谱 API (`/api/graph/`)

```
GET  /stats                     图谱统计（节点数/边数/名人列表）
GET  /data?filter=celebrity     图数据（D3 node-link 格式）
GET  /person/{name}             名人详情（关系、事件、graph_context）
GET  /path?from=X&to=Y          两人间最短路径
GET  /impact?name=X&event=Y     事件对某人的影响评估
GET  /celebrities               所有名人名字列表

### 危机仿真 API (`/api/crisis/`)

```
GET  /scenarios                             场景列表
GET  /scenario/{title}                      单场景详情
POST /start                                 启动仿真（body: {scenario_title, total_days}）
POST /step                                  推进一天
POST /run                                   运行到底（body: {days}）
POST /intervene                             添加干预
GET  /state                                 当前状态
GET  /history                               完整历史
GET  /outcome                               结果分析报告
POST /reset                                 重置
GET  /celebrities                           名人列表（供自由组局）
GET  /relationships?persons=a,b,c           查询关系
POST /custom-scenario                       创建自定义场景并启动
POST /experiment/create                     创建 A/B 实验
POST /experiment/{id}/run                   运行实验
GET  /experiment/{id}/compare               获取对比结果
GET  /experiments                           列出所有实验

### 综艺仿真 API (`/api/sim/`)

```
GET  /state                    仿真状态（环境、Agent、反应系统）
GET  /history?limit=20         最近动作历史
GET  /snapshots                反应系统时间序列
POST /start                    启动综艺仿真
POST /pause                    暂停
POST /resume                   继续
POST /stop                     停止
POST /inject                   注入外部事件
```

## 十、常见问题

**Q: 需要 API Key 吗？**
不需要。所有功能都有纯规则引擎 fallback。配置 LLM API Key 后 Agent 决策更自然，但不影响核心功能。

**Q: 为什么图谱只有 10 个明星？**
数据来自 `celebrity_scraper/data/` 目录下的 JSON 文件。可以运行爬虫添加更多：
```bash
python test_scraper.py real 肖战
python test_scraper.py real 杨幂
```

**Q: 自由互动模式和危机模式有什么区别？**
危机模式有 6 阶段生命周期（爆发→发酵→高峰→应对→收尾→余波），Agent 选择 10 种 PR 策略。自由互动没有阶段，Agent 选择 11 种日常动作（发言/支持/批评/合作等）。

**Q: A/B 实验如何工作？**
同一场景分别用不同干预条件运行多次仿真，系统自动对比各组口碑分、热度、品牌价值等指标，找出最佳策略。

**Q: 如何调试？**
```bash
# 查看完整仿真日志
python -m demos.crisis_simulation 2>&1 | tee sim.log

# 测试单个 API
curl -s http://localhost:8765/api/crisis/scenarios | python -m json.tool
```
